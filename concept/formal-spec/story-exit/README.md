# Story Exit

Dieser Kontext formalisiert den administrativen Exit aus einer laufenden
Story-Execution in einen menschlich gefuehrten `ai_augmented`-Modus.

Er deckt insbesondere ab:

- leichte menschliche Exit-Freigabe
- nicht-resumable Run-Terminalitaet
- Story-Cancel mit `viability_handoff`
- Cleanup von Lock, Bindung und Story-Regime
