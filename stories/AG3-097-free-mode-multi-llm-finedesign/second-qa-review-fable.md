## Fixed findings — before/after

- **F1** `client.py:_answered_flag` — before: `return _int_value(row.get("message_count")) > 0` (sends invent an answer); after: `return False` when neither `answered` nor `response_count` is present (fail-closed; `response_count` derivation kept — it is a real answer fact). Test: `tests/unit/multi_llm_hub/test_multi_llm_hub_client.py::test_session_stats_never_derives_answered_from_sent_message_count` (5 sends, no flag/no response_count -> `answered=False`; `response_count: 3` -> True; `response_count: 0` -> False).
- **F2** `exploration/phase.py` — before: `_run_fine_design` returned `None` on converged and the gate/freeze used the original frame (decisions telemetry-only); after: it returns the frame enriched via `model_copy(update={"fine_design_decisions": outcome.final_design_decisions})`, threaded through `_MandateRouting.change_frame` into the gate + freeze, so `DesignFreezeMarker` persists the decision protocol into the protected `change_frame.json` (single writer kept — no second write path, freeze behaviour untouched). Tests: `tests/unit/exploration/test_phase_mandate.py::test_converged_fine_design_decisions_are_persisted_into_frozen_frame` (frozen successor carries FD-001; telemetry still fires) and the real-path `tests/integration/exploration/test_phase_full.py::test_converged_hub_fine_design_persists_decisions_into_frozen_artifact` (full productive assembly: `build_hub_fine_design_evaluator` + real `LlmConvergenceJudge` over a scripted verdict completion + real review/freeze/state-backend -> frozen artifact JSON contains `fine_design_decisions`, hub really driven + released).
- **F3** `hub_fine_design.py:_warn_on_bad_release` docstring — before: "reads the post-hoc stats BEFORE this evaluator's own release call"; after: "stats are read AFTER this evaluator's own release call (see finalize): a session the hub still reports as not released at that point is a real release violation."
- **F4** `hub_fine_design.py:_acquire_advisors` — before: only `_send_counts` reset on fresh acquisition; after: also `self._last_responses = {}` (a fresh acquisition starts a fresh discussion). Test: `tests/unit/exploration/mandate/test_hub_fine_design.py::test_retry_attempt_starts_with_fresh_previous_responses` (recording prompt builder: attempt-2 round-1 sees `{}`, attempt-1 round-2 saw positions).
