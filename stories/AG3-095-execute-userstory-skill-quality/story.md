4. **`SkillQualityMetric` + `collect_quality_metrics` (§43.6.2, fail-closed Skeleton):** Sub `skills/quality_metric/` real bauen; `SkillQualityMetric` als typisiertes Pydantic-v2-Modell mit den unten definierten Feldern. `collect_quality_metrics(skill_name, *, project_key, source_window)` liest ausschliesslich via `Telemetry.ProjectionAccessor.read_projection(...)` (Telemetrie-Projektionen + `FC_INCIDENTS`-Projektion) und aggregiert die quellseitig **heute** verfuegbaren Signale.    **`SkillQualityMetric`-Schema (englische Identifier/Keys, ARCH-55):**
   - Filter/Identitaet: `skill_name: str`, `project_key: str`, `source_window: SourceWindow` (typisiertes Zeitfenster), `bundle_version: str | None` (fail-closed `None`, solange quellseitig nicht zuordenbar).
   - Telemetrie-Aggregate (aus Projektionen): `usage_count: int`, `successful_runs: int`, `failed_runs: int`, `unknown_status_runs: int`, `avg_qa_rounds: float | None`, `remediation_count: int`.
   - Failure-Corpus-Aggregate (aus `FC_INCIDENTS`): `incident_count: int`, `incident_ids: tuple[str, ...]`.
   - Attributions-Status: `attribution: AttributionState` (StrEnum `ATTRIBUTED` | `UNATTRIBUTABLE`), gesetzt `UNATTRIBUTABLE`, solange die Quelle keine Skill-/Version-Zuordnung traegt (siehe Ist-Zustand).
   - **Deterministische `final_status`-Klassifikation (kein Free-String-Durchreichen):** Jeder Projektionseintrag traegt ein `final_status` als freien String (`StoryMetricsRecord.final_status`, `src/agentkit/closure/post_merge_finalization/records.py:23`; im Lauf u.a. `"completed"`/`"failed"`/`"escalated"`/`"blocked"`/`"yielded"` aus `PipelineRunResult.final_status`, `src/agentkit/pipeline_engine/runner.py:39-41`, sowie terminale `COMPLETED`/`DONE`/`MERGED`/`CLOSED`). `collect_quality_metrics` normalisiert jeden Wert via `final_status.strip().upper()` und klassifiziert anhand **zweier typisierter, geschlossener Mengen** (modul-konstante `frozenset`, keine Inline-String-Kaskaden):
     - **Success-Set** = die bereits existierende Terminal-Erfolgsmenge `_COMPLETED_STATUSES` (`{"COMPLETED", "DONE", "MERGED", "CLOSED"}`, `src/agentkit/telemetry/audit_bundle.py:50`) — diese **bestehende** Konstante ist die Single-Source-of-Truth fuer „erfolgreicher Run" und wird wiederverwendet/importiert, **nicht** neu erfunden (FIX-THE-MODEL).
     - **Failure-Set** = `{"FAILED", "ESCALATED", "BLOCKED"}` (terminale Fehl-/Abbruch-Status aus `runner.py`).
     - **Unknown/fail-closed:** jeder Wert, der in **keiner** der beiden Mengen liegt (z. B. `"YIELDED"` = nicht-terminal, oder eine unbekannte Schreibweise), zaehlt **nicht** als Erfolg und **nicht** als Fehler, sondern wird in `unknown_status_runs` gezaehlt. Unbekannte Status werden **nie** stillschweigend einem der beiden Buckets zugeschlagen (FAIL-CLOSED, kein Schoengen).
   - Aggregationsregeln:
     - `usage_count` = Anzahl der ins Fenster + `project_key` (+ kuenftig `skill_name`, siehe Attribution) fallenden `StoryMetricsRecord`-Projektionseintraege.
     - `successful_runs` = Anzahl Eintraege mit normalisiertem `final_status` im Success-Set; `failed_runs` = im Failure-Set; `unknown_status_runs` = sonst. Es gilt invariant `successful_runs + failed_runs + unknown_status_runs == usage_count` (Test-Assertion).
     - Erfolgsquote = `successful_runs / usage_count` (Nenner = `usage_count`, fail-closed `None` bei `usage_count == 0`).
     - `avg_qa_rounds` = arithmetisches Mittel der `qa_rounds` ueber das Fenster (`None` bei leerem Fenster).
     - **`remediation_count` deterministisch aus `qa_rounds`:** `qa_rounds` = Anzahl Implementation-QA-Versuche (`len(load_attempts(story_dir, "implementation"))`, `src/agentkit/closure/post_merge_finalization/metrics.py:67`). Der erste Versuch ist kein Remediation-Versuch; jede weitere Runde ist eine Remediation. Pro Eintrag gilt `max(qa_rounds - 1, 0)`; `remediation_count` = Summe dieser Werte ueber das Fenster (`0` bei leerem Fenster, **kein** `None` — Wert ist quellseitig stets ableitbar).
     - `incident_count`/`incident_ids` aus `FC_INCIDENTS` gefiltert auf Fenster + `project_key`.
